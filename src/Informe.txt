Este programa en java funciona como un sistema basico para registrar productos tecnologicos y calcular sus impuestos. El objetivo es guiar al usario por una serie
de pasos donde debe ingresar información del producto y, con esos datos el programa realiza distitons cálculos, todo se maneja desde la consola y cada dato ingresado 
se revisa para evitar errores.


Inicio de flujo principal:
Al comenzar, el sistema solicita el numero maximo de productos que se van a registrador. Despues pide el dia de operación. Esto es importante porque algunos impuestos
solo apllican en fin de semana, para evitar valores incorrectos, el programa repite la pregunta hasta recibir un numero valido que es entre el 1 y el 7.


Resgistro de cada producto:
Cuando el sistema ya tiene esa información, entra en un ciclo que se repite tantas veces como productos el usuario haya indicado. Para cada prodcuto se pide el nombre,
el precio base, la categoria, el riesgo tecnológico y el stock. cada dato se valida para asegurarse de que tenga sentido, como que el precio sea mayor que cero o que
la categoria este entre el 1 y el 4


Manejo de categorias:
La categoria es un punto importante ya que influye en los impuestos que se aplican, por ejemplo, la categoria 3  esta exenta del IVA. También se lleva un recuento de 
cuántos productos pertencen a cada categoría para determinar cuál fue la mas ingresada al final del proceso.


Cálculo de impuestos:
Después de recibir los datos del producto, el programa calcula los impuestos necesarios. El IVA se aplica si la categoria lo permite, el IMC siempre se calcula, el IRT
se aplica cuando el producto tiene riesgo tecnológico, el TCE se suma únicamente si el registro se realiza los sabados o los domingos, con todos estos valores el sistema
obtiene el precio final del producto y lo muestra al usario.


Acumulación de resultados:
Mientras procesa los productos, el programa acumula los valores totales de cada impuesto. También identifica el producto con el precio final más alto para resaltarlo en
el informe final.


Generación del reporte general del dia:
Cuando se han registrado todos los productos, el sistema presenta un resumen que incluye el total de productos ingresados, los montos recaudados por cada impuesto, el monto
total del dia, la categoria mas registrada y el producto mas costoso.


Conclusión:
En conjunto este programa demuestra cómo manejar ciclos, validaciones, cálculos y acumulación de datos dentro de un sistema sencillo, aunque tampoco es un sistema complejo,
muestra claramente como se puede organizar la informacion de varios productos y asi generar resultados precisos.
